第一块：组件架构

现在的 Chat.vue 有 1326 行，所有逻辑混在一起。新设计需要拆分

重构后的组件树
ChatPage.vue                    ← 顶层：布局容器 + 模式切换逻辑
├── Sidebar.vue                 ← 会话列表 (已有，基本不改)
├── ChatPanel.vue               ← 左栏：聊天消息 + 输入框
│   ├── MessageBubble.vue       ←   单条消息气泡 (用户/AI)
│   ├── StepProgress.vue        ←   内联步骤进度 (已有)
│   └── InputBar.vue            ←   底部输入组件
├── ThinkingPanel.vue           ← 中栏：思考过程 (新增)
│   └── 流式文本 + 步骤卡片
└── ResultPanel.vue             ← 右栏：结构化结果 (改造 RightPanel)
    ├── ToolCallCard.vue        ←   工具调用卡片
    ├── PlanBoard.vue           ←   执行计划看板
    ├── RouteMap.vue            ←   地图路线 (已有)
    ├── PoiList.vue             ←   POI 列表
    └── PdfDownload.vue         ←   PDF 下载

相比现在的改进

  • Chat.vue 1326行 → ChatPage.vue ~150行 (容器)
  • 每个面板独立开发、独立测试
  • SSE 事件路由逻辑集中在 ChatPage，各面板只接收自己的数据
  • 可复用：ResultPanel 就是 RightPanel 的升级版

需要注意

  • 组件间通信：兄弟组件不直接通信，通过 ChatPage 中转
  • 状态管理：如果不用 Pinia，用 provide/inject 或 props down / events up
