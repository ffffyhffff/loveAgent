第二块：SSE 事件路由 & 数据流

目前所有 SSE 事件都在 Chat.vue 里处理，现在要分发给三个面板

事件分发规则
SSE Event      →  ChatPanel    ThinkingPanel   ResultPanel
─────────────────────────────────────────────────────────
text           →  ✅ 气泡追加    —               —
plan / step    →  —            ✅ 步骤卡片       ✅ 计划看板
section        →  —            —               ✅ POI 列表
review         →  —            —               ✅ 详情卡片
placeDetail    →  —            —               ✅ 地点详情
pois           →  —            —               ✅ POI 选择器
map            →  —            —               ✅ 地图路线
pdf            →  —            —               ✅ 下载卡片
form           →  ✅ 表单提示    —               —
done           →  ✅ 完成标记    ✅ 完成标记      ✅ 完成标记
error          →  ✅ 错误提示    ✅ 错误提示      ✅ 错误提示

模式切换逻辑

简单对话模式：收到第一个 plan 或 step 事件前，只显示 ChatPanel。
Agent 执行模式：一旦收到 plan 事件 → 自动展开三栏，ThinkingPanel + ResultPanel 切入。
回退：done 事件后保持展开，用户可手动收起。下一次对话开始时重置。

状态管理方案

不加 Pinia，用 Vue 3 的 provide/inject：
ChatPage 用 reactive() 维护共享状态，通过 provide() 注入。子组件用 inject() 读取。
或者简单用 props down / emits up，因为层级只有 1-2 层。
