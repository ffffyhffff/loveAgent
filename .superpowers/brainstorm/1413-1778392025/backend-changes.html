🆕 建议新增

thinking — 思考流式文本

Agent 内部的推理文字，如："分析用户意图... 拆解为3个子任务..."

目前 qwen-plus 不暴露原生 CoT，但可以：

  • 在后端 Agent 的 prompt 中要求先输出推理再调用工具
  • 把工具调用前的文本以 thinking 事件发送
  • 或直接用 step.message 作为思考内容

推荐：先用 plan.steps[].message + 每个 step 的描述作为思考内容，零后端改动先跑通，后续再优化。

最终方案：零后端改动启动

ThinkingPanel 展示 plan.steps[] + step 事件的 message 字段，流式追加到"推理链"区域。每个 step 一条带左侧竖线的思考块。
ResultPanel 已有数据结构完全够用——plan/step 喂计划Tab，section 喂工具Tab，map/pois 喂地图Tab，pdf 喂PDF Tab。
ChatPanel 只消费 text 事件，保持传统气泡体验。用户输入区下方加一个"当前状态"小字提示（如"AI 正在规划中..."）。
